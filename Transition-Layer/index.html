第5章 传输层

5.1 传输层提供的服务

传输层为应用层提供可靠通信服务

5.1.1 传输层的功能

  1. 提供端到端的通信
  2. 复用和分用
    • 复用：发送方的不同应用进程均可使用同一传输层协议传送数据
    • 分用：接收方的传输层可以在剥离报文首部后把这些数据正确交付到目的应用进程
  3. 对报文的首部以及数据部分进行差错检测（如何对数据进行检错？）
  4. 同时提供两种不同传输协议
    1. 面向连接的TCP（无论底层是否可靠，逻辑上就是可靠的信道）
    2. 无连接的UDP（逻辑上是不可靠信道）

5.1.2 传输层的寻址与端口

  1. （软件）端口的作用

    • 用于唯一标识主机中应用进程的接口（根据ip地址和mac地址类推，但是感觉有些不对？）
    • 是应用层的各种协议进程与传输实体进行层间交互的一种地址

  2. 端口号

    • 端口号长度为16位，最多能够表示65536个端口号，只具有本地意义

    • 服务端使用的端口号

      • 熟知端口号 数值为0~1023 被指派给TCP/IP最重要的一些应用程序

      • 登记端口号 数值为1024~49151

        • 应用程序 FTP TELNET SMTP DNS TFTP HTTP SNMP
          熟知端口号（相对于1024的ofs） 21 23 25 53 69 80 161

    • 客户端使用的端口号

      • 数值为49152~65535
      • 一个进程在通信结束后就释放占用的端口号，这个端口号就可被其他进程使用

  3. 套接字Socket=IP Address:Port

5.2.2 UDP校验

在计算校验和时，要在UDP数据报之前增加12B的伪首部，伪首部仅用于计算校验和

伪首部的格式： |源ip地址（4B）|目的ip地址（4B)|0(1B)|17(1B)|UDP长度（2B)|

校验过程：

  1. 在发送端
    1. 伪首部，首部，数据段分为每段2B长的数据，数据段若不足则填充0
    2. 对上述各段数计算求和,进位也加在最低位，最终结果按位取反
    3. 结果填入检验和字段
    4. 去掉伪首部，发送
  2. 在接收端
    1. 填上伪首部
    2. 同样的求和取反码
    3. 结果为全1则无差错，否则进行差错处理
