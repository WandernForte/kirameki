计算机网络体系结构

2023年05月09日 2.5k 字 大概 10 分钟

1.1 计算机网络概述

1.1.1 计算机网络的概念

计算机网络就是若干互联、自治的计算机系统的集合

三种定义观点

  1. 广义观点
    • 远程资源共享就是计算机网络
    • 是最原始的观点
  2. 资源共享观点
    • 符合目前计算机网络的特点
    • 目的：资源共享
    • 组成单元：互联但自治的若干计算机系统
    • 资源共享统一规则：网络协议
  3. 用户透明性观点
    • 网络未来发展目标
    • （现在没啥意义就是说）

世界上最早的计算机网络是ARPAnet

1.1.2 计算机网络的组成

  1. 从组成成分上看
    • 硬件
      • 主机
      • 通信链路
      • 交换设备
      • 通信处理机
    • 软件
    • 协议
  2. 从工作方式上看
    • 边缘部分
      • 主机
    • 核心部分
      • 路由器，负责连通与交换
  3. 功能组成上看（同时也是计算机网络系统的基本组成）
    • 通信子网
      • 传输介质，通信设备，相应网络协议
      • 网桥、交换机、路由器
      • 是决定网络拓扑结构的主要因素
    • 资源子网
      • 实现资源共享的设备及其软件的集合
      • 计算机软件，进程

1.1.3 计算机网络的功能

  1. 数据通信
  2. 资源共享
  3. 分布式处理
  4. 提高可靠性，提高工作效率
  5. 负载均衡

1.1.4 计算机网络的分类

  1. 按分布范围（降序）

    1. 广域网WAN 使用交换技术
      • 拓扑结构通常采用网状结构
    2. 城域网MAN 使用以太网技术
      • 拓扑结构通常是环形结构或者网状结构
    3. 局域网LAN 使用广播技术
      • 局域网和广域网之间通过路由器进行互联，网关将局域网的数据转换为适合在互联网上传输的数据格式和协议
      • 拓扑结构通常采用星形
    4. 个人区域网PAN

这些类别的差异主要体现在覆盖范围以及使用的技术协议上

  1. 按传输技术

    1. 广播式网络
      • 所有联网计算机共享一个公共通信信道（共享信道）
      • 不存在路由选择->可以不需要网络层
    2. 点对点网络

  2. 按拓扑结构

    • 拓扑结构是指网中结点与通信线路之间的几何关系表示的网络结构
    • 总线形
    • 星形
    • 环形
    • 网状

  1. 按使用者

    1. 专用网
    2. 共用网

  2. 按交换技术

    1. 电路交换网络
      1. 在源节点和目的结点之间建立一条占用的通路用于传输数据，包括建立连接，传输数据和断开连接三个阶段。
      2. 优点：数据直接传输，时延小
      3. 缺点：线路利用率低，不便于进行差错控制
    2. 报文交换网络（存储-转发网络）
      1. 报文：用户数据，源地址，目标地址，校验码等辅助信息的封装
      2. 特点：整个报文到达相邻结点，全部存储后（就是接收成功，存储在结点的存储空间中，与全部不全部无关联，怎么翻译的这是），查找转发表，转发到下一个节点
      3. 优点：充分利用线路，实现不同链路不同传输速率的转换，可以实现格式转换，可以实现一对多，多对一的访问，可以实现差错控制
      4. 缺点：增大资源开销（每个结点都要存储一份报文的副本，缓冲区管理难度大，以及封装操作的开销），增加了缓冲时延
    3. 分组交换网络
      1. 将数据分成较短的定长的数据块，在每个数据块中加上目的地址、源地址等辅助信息组成分组（或包），以存储-转发方式传输
      2. 优点：缓冲易于管理，平均时延更小，更易于标准化(相对于报文交换改进的直接结果就是减少了传输时延)

  3. 按传输介质

    1. 有线
    2. 无线

1.1.5 计算机网络标准化工作（略）

1.1.6 计算机网络的性能指标

  1. 带宽 bit/s(bps)
  2. 时延
    • 发送时延（HIT版为传输时延）=分组长度/信道带宽
      • 是结点将分组的所有bit推向传输链路所需的时间
    • 传播时延=信道长度/电磁波在信道上的传播速率（一般由题目给出）
    • 处理时延（一般忽略不计）
    • 排队时延（一般忽略不计）
    • 转发时延 端口状态改变的延迟。例如，端口从空闲状态转变为工作状态的时延
  3. 时延带宽积=传播时延*信道带宽 表示发送方在收到一个响应前能够发送的数据量

    时延带宽积在后续处理发送间隔将会频繁用到

  4. 往返时延
  5. 吞吐量
  6. 速率
  7. 信道利用率=有数据通过时间/总时间

1.2 计算机网络体系结构与参考模型

1.2.1 计算机网络分层结构

计算机网络的各层及其协议的集合称为网络的体系结构

1.2.2 计算机网络协议、接口、服务的概念

  1. 协议
    • 语法：定义传输格式
    • 语义：规定所要完成的功能
    • 同步：规定时序关系
    • 功能：线路管理，差错控制，数据转换
  2. 接口
    • 服务访问点（SAP)：跨层次访问的逻辑接口
  3. 服务**[下层为紧邻的上层提供的功能调用]**
    • 服务原语
      • 请求 服务用户方->服务提供方
      • 指示 提供方->用户方
      • 响应 用户方->提供方
      • 证实 提供方->用户方
    • 服务分类
      • 面向连接服务与无连接服务
        • 面向连接服务双方必须先建立连接，分配相应的资源。例如TCP协议
        • 无连接服务双方无需先建立连接，数据通过线路传输，不可靠。例如IP，UDP协议
      • 可靠服务与不可靠服务
        • 是否具有纠错能力
      • 有应答服务与无应答服务
        • 有应答服务的实例：文件传输协议 ftp

1.2.3 ISO/OSI参考模型和TCP/IP模型

为什么需要分层？

  • 简化网络通信过程
  • 提高可靠性
  • 支持多种协议，增加功能之间的独立性
  • 促进标准化，包括定义标准界面以及提供标准语言
  • 方便管理和维护等方面

OSI参考模型

  1. 物理层

    1. 定义数据终端设备和数据通信设备的物理与逻辑连接方法，传输原始比特流（节点到节点）
    2. 不参与数据封装，因为数据封装是当前层将数据传输给下一层必须要
    3. 接口协议：EIA-232C、EIA/TIA RS-449、CCITT的X.21等
    4. 服务访问点：网卡接口

  2. 数据链路层

    1. 任务是将网络层传来的IP数据封装成帧，检测差错并进行处理。即数据接收与发送。（设备到设备，节点到节点）

    2. 功能：

      1. 成帧（定义数据格式），保证数据正确的顺序和完整性
      2. 差错控制，流量控制，传输管理（物理寻址）（提供了结点间的可靠传输），
      3. 链路管理（控制对物理传输介质的访问）

    3. 接口协议：SDLC,HDLC,PPP（SILP）,STP和帧中继等

    4. 在不可靠的物理介质上提供可靠的传输–存在纠错

    5. 服务访问点：MAC地址（网卡地址）

  3. 网络层

    1. 任务是把网络层的协议数据单元从源端传到目的端，分发数据分组（起路由作用，主机到主机，节点到节点）
    2. 功能：流量控制，拥塞控制，差错控制，网际交互
    3. 可提供有连接与无连接服务
    4. 接口协议：IP、IPX、ICMP,ARP,RARP,OSPF
    5. 服务访问点：IP地址（网络地址）

  4. 传输层

    1. 负责不同主机的进程之间的通信，包括建立、维护、拆除等操作端到端
    2. 差错控制，流量控制，拥塞控制，服务质量，数据传输管理
    3. 接口协议：TCP,UDP
    4. 服务访问点：端口号

  5. 会话层

    1. 校验，管理主机间的会话进程（端到端）

  6. 表示层

    1. 数据格式转换，设计数据封装（端到端）

  7. 应用层

    1. 端到端
    2. 接口协议：FTP,SMTP,HTTP
    3. 服务访问点：用户界面

TCP/IP模型

提供无连接不可靠的数据报（文）服务，包含TCP、IP、UDP、Telnet、FTP、SMTP、ICMP等协议

  1. 应用层
    1. 包含所有高层网络协议
  2. 传输层
    1. 接口协议：TCP,UDP（与OSI模型类似）
  3. 网际层
    1. 定义了标准的分组格式和协议
  4. 网络接口层
    1. 与物理网络的接口

考试中涉及的是OSI模型（从高到低）

  • 应用层
  • 传输层
  • 网络层
  • 数据链路层
  • 物理层

考试中需要熟记各个层次的功能

  1. 物理层（Physical Layer）：负责在物理媒介上传输比特流（0和1），并定义物理层面的电气特性、连接器和传输速率等参数。
  2. 数据链路层（Data Link Layer）：负责将比特流组装成帧（Frame），并为每一帧添加标识和校验等信息，以确保数据的可靠传输。
  3. 网络层（Network Layer）：负责进行路由选择和流量控制等工作，以确保数据在网络中的正确传输和到达目标地址。
  4. 传输层（Transport Layer）：提供端到端的可靠数据传输服务，负责数据分段、数据重组、数据传输可靠性的保证，以及控制数据的流量和拥塞等问题。
  5. 应用层（Application Layer）：为用户提供各种网络应用服务，如电子邮件、文件传输、远程登录、Web服务等。
