计网：数据链路层

2023年05月14日 4.8k 字 大概 20 分钟

3.1 数据链路层的功能

3.1.1 为网络层提供服务

数据链路层不关心与物理层的实现，以及对物理层的作用。

例如，数据链路层不会对终端节点隐蔽物理传输的细节，因为它根本就不关心

  1. 无确认的无连接服务
    • 规则：源机器发送数据帧时不需先建立链路连接，目的机器收到数据帧时无需发回确认。
    • 差错处理：对于丢失的帧，数据链路层不负责重发而交给上层处理。
    • 适用范围：可靠的实时通信信道或误码率较低的通信信道，如以太网
  2. 有确认的无连接服务
    • 规则：源机器发送数据帧时不需先建立链路连接，目的机器收到数据帧时必须发回确认。
    • 差错处理：源机器在规定时间内未收到确定信号，就重传丢失的帧
    • 适用范围：误码率较高的通信信道，如无线通信
  3. 有确认的面向连接服务
    • 规则：建立数据链路、传输帧、释放数据链路。目的机器对收到的每一帧都要给出确认，源机器收到确认后才能发送下一帧
    • 适用范围：实时可靠的通信要求（通信信道不一定实时可靠）
    • 不存在无确认的面向连接服务，因为有连接就肯定有确认

3.1.2 链路管理

数据链路层连接的建立、维持和释放过程称为链路管理。

3.1.3 帧定界、帧同步与透明传输

HDLC协议标准帧格式：

|标志|地址（8）|控制（8）|信息|帧校验序列（16）|标志|[标志固定为01111110]

[数据链路层协议]：SDLC, HDLC, PPP（SILP）, STP和帧中继等)

3.1.4 流量控制

一般在传输层讨论流量控制

3.1.5 差错控制

  1. 循环冗余校验（CRC）用于发现位错
  2. 自动重传请求（ARQ）用于重传出错的帧
  3. 校验和

3.2 组帧

封装网络层数据包构成数据帧

3.2.1 字节计数法

  • 在帧头部使用一个计数字段来标明帧内字节数，字节数包含自身这个字节
  • 计数字节出错将无法补救

3.2.2 字节填充的首尾定界法

字符填充法使用特定字节来定界一帧的开始（SOH0x7E）与结束(EOT)，转义字节ESC

3.2.3 零比特填充的首尾计数法

以01111110作为开头与结尾的标志，数据域每5个1插入一个0

3.2.4 违规编码法

适用于冗余编码的环境

3.3 差错控制

3.3.1 检错编码

  1. 奇偶校验码
    1. 无法检测到偶数个出错位时的错误
  2. 循环冗余码（CRC)
    1. 设定一个多项式G
    2. 假设一个帧对应多项式为M(x),则计算冗余码的步骤如下
      1. 在M(x)的二进制系数低位后加G(x)阶数r个0，得到H(x)；
      2. 模2除。H模2除以G，所得余数即为冗余码(校验码)
      3. M与冗余码拼接就得到了发送的数据X
      4. 接收方通过将X模2除以G看是否有余数来确定数据是否出错
  3. 校验和
    1. 将数据划分为各个元素长16位的序列
    2. 各个元素进行补码求和得到sum
    3. 取sum反码得到校验和插入数据（头or尾？）
    4. 接收方将收到的数据+校验和信息再进行分组&补码求和计算，为16位全0或全1时无差错

3.3.2 纠错编码

汉明码

计组，不赘述，只补充计组不考的部分

汉明码纠错d位，需要码距为2d+1的编码方案；检错d位需要d+1位

3.4 流量控制与可靠传输机制

3.4.1 流量控制，可靠传输，滑动窗口机制

停止-等待流量控制

发送方每发送一帧，都要等待接收方的应答信号，之后才能发送下一帧

滑动窗口流量控制

发送方维持一组连续的允许发送的帧的序号，称为发送窗口；同时接收方也维持一组连续的允许接收帧的序号。

重要特性

  1. 接收方发送回确认帧且发送方接收到的话，发送窗口才会向前滑动
  2. 窗口大小决定了几种不同的传输协议
    1. 停止-等待协议：发送窗口=接收窗口=1；
    2. 后退N帧协议：发送窗口>1,接收窗口=1；
    3. 选择重传协议：发送窗口>=接收窗口>1;$且发送窗口大小+接收窗口大小\leq2^n$
  3. 接收窗口大小为1时，可保证帧的有序接收；
  4. 数据链路层的滑动窗口协议中，窗口的大小在传输过程中是固定的

可靠传输机制

确认+重传+序号标识+校验+计数器

3.4.2 单帧滑动窗口与停止-等待协议

除数据帧丢失外的两种差错

  1. 帧损坏
  2. 确认帧损坏
  3. 对数据帧和确认帧添加1bit编号，连续出现相同序号的数据帧，说明发送端进行了超时重传。连续出现相同序号的确认帧，说明接收端接收到了重复帧。这种方法要求双方均保存一个帧的副本以进行比对。（Rdt2.1）

3.4.3 多帧滑动窗口与后退N帧协议（GBN）

接收方只允许按顺序接收帧，同时可以在连续接收到多个正确的数据帧后才对最后一帧发送确认帧。

即，对某一数据帧的确认代表该数据帧和此前所有数据帧均已正确收到。

发送窗口大小：若采用n位对帧编号，则发送窗口的大小Wt应满足$ 1\leq W_t\leq 2^n-1$

Ps:在信道的误码率较大时，由于多次重发，效率可能劣于停止-等待协议

3.4.4 多帧滑动窗口与选择重传协议（SR)

只重传出现差错的数据帧或计时器超时的数据帧,是一种无序接收的滑动窗口协议

窗口大小：双方的窗口大小W应满足$1<W\leq 2^{(n-1)}$

3.5 介质访问控制

为每个节点隔离来自同一信道上其他节点所传输的信号

3.5.1 信道划分（方法）介质访问控制

多路复用（静态信道划分）

静态信道划分没有碰撞发生的风险

  1. 频分多路复用（FDM)
    1. 频分多路复用的各用户占用不同的带宽资源（请注意，这里的 “带宽”是载波频率的域（单位：Hz）而不是数据的发送速率）
    2. 用户在分配到一定的频带后，在通信过程中自始至终都占用这个频带
    3. 特点：充分利用传输介质的带宽，系统效率较高，技术成熟
  2. 时分多路复用（TDM）
    1. 将一个时间段分成若干时间片，轮流分配给多个信号使用
    2. 特点：抗干扰能力强，介质的位速率大于单个信号的位速率，可以传输数字信号
  3. 波分多路复用（WDM）
    1. 一束电磁波的不同频分分配给多个信号使用
  4. 码分多路复用（CDM）
    1. 广泛应用于无线链路共享（如蜂窝网，卫星通信等）
    2. 每个用户分配一个唯一的m bit码片序列，其中“0”用”-1”表示，“1”用“+1”表示。各用户使用相同频率的载波
    3. 编码信号=原始数据x码片序列
      1. 即：发送1则发送m bit码片序列原码
      2. 发送0则发送 m bit码片序列的反码
    4. 各用户码片序列需要相互正交
    5. 解码：求码片序列与编码信号的内积，即可分理出所需的信号

3.5.2 随机访问（协议）介质访问控制

动态划分信道，有碰撞的风险，需要设计检测碰撞策略来避免/解除碰撞

ALOHA协议

  1. 纯ALOHA协议

    1. 基本思想：当网络中的任何一个站点需要发送数据时，可以不进行任何检测就发送数据。如果超时，就认为是在传输过程中发生了冲突。发送站点就将在一段随机的时延后重新发送数据，直至发送成功。

    2. 吞吐量为0.184

  2. 时隙ALOHA协议

    1. 吞吐量0.368

    2. 同步各站点的时间，不同的站点分配不同的时隙（类似于时间片，这个时隙足够发送一个数据帧）。如果发生冲突，则在下一个时隙开始时以概率重发数据帧。

    3. 优点：

      1. 单个结点活动时，可以连续以信道全部速率传输（？）
      2. 只需同步时隙，高度分散化
      3. 简单

    4. 缺点：

      1. 不能消除冲突，浪费时隙
      2. 存在空闲时隙，浪费时隙

CSMA协议

  1. 1-坚持CSMA:
    1. 基本思想：一个节点要发送数据时，首先侦听信道如果信道空闲，那么必然立即发送数据；如果信道忙，则不发送，继续坚持侦听；如果信道发生冲突，那么随机等待一段时间再重新开始侦听。
  2. 非坚持CSMA:
    1. 与1-坚持不同的是，如果信道忙则放弃监听并在随机等待一段时间后继续侦听，（放弃，侦听，）直到信道空闲。
  3. p-坚持CSMA:
    1. 基本思想：一个节点要发送数据时，首先侦听信道如果信道空闲，那么以p的概率立即发送数据，否则等待到下一个时隙，此期间继续坚持侦听；如果信道发生或忙碌，那么随机等待一段时间再重新开始侦听。

CSMA/CD协议

“先听后发，边听边发，冲突停发，随机重发”

争议期（也称争用期）：信号最远两个端点之间往返传输的时间, 即两次单向传播时延

二进制指数退避算法

  1. 确定基本退避时间，一般取两倍的总线端到端传播时延2t
  2. 定义参数k=min{重传次数, 10}，
  3. 从整数集$[0,1,…,2_k-1]$中随机取一个数r重传所需要的退避时间就是2rt
  4. 重传16次仍未成功时放弃重传，并向上层报告出错
  5. 重传次数越多，越能降低发生碰撞的概率

最短帧长计算

$最小帧长=总线传播时延\times 数据传输速率\times 2$

极限最短帧长为64B

最长帧长为1518B(1500B数据+18B头部)

CSMA/AC协议

算法流程

  1. 若站点发送第一个数据帧，且检测到信道空闲，在等待时间DIFS后，发送整个数据帧
  2. 否则（重传的情况或信道忙碌），站点执行CSMA/AC退避算法（算法内容不重要，只需要知道有个随机倒计时决定是否重发），选取一个随机回退值。一旦检测到信道忙退避计时器就保持不变，一旦空闲就开始倒计时。
  3. 当退避计时器为0时，重发并等待确认

对信道进行预约

先广播一个RTS(请求发送)帧，通知到范围内的所有站点。如果信道空闲，目的站点就广播一个CTS(允许发送)帧，到范围内所有的站点。刚才交换了RTS和CTS的两个站点开始建立传输。

3.5.3 轮询访问：令牌传递协议

  1. 等待令牌
  2. 接收令牌&&发送数据（通过接收到的令牌的数据块封装数据帧）&&重新生成令牌，发给下一个可以使用信道的结点
  3. 最后发回到发送方检测是否出现差错。
  4. 理论上不会发生碰撞或冲突，可以承担较大的网络负载

3.6 局域网

3.6.1 局域网的基本概念和体系结构

三要素

  1. 拓扑结构
    1. 星形
    2. 环形
    3. 总线形
    4. 星形x总线形
  2. 传输介质
  3. 介质访问控制方式
    1. CSMA/CD 用于总线形拓扑结构
    2. 令牌总线 用于总线形拓扑结构
    3. 令牌环 用于环形局域网等负载较重的网络环境

3.6.2 以太网与IEEE 802.3

  1. 以太网用两种方式简化通信
    1. 无连接的工作方式，提供不可靠服务(有线介质，链路相对可靠)，对于差错的纠正由高层完成。
    2. 发送的数据使用曼彻斯特编码，接收端利用电压转换的特性把位同步信号提取出来
  2. 以太网的传输介质与网卡
    2. 网卡：进行数据的串并（合并的意思）转换
    3. 中继器：加强基带信号，延长网络所支持的长度
  3. 以太网的MAC帧
    1. MAC地址长6字节（48位），高24位为厂商代码，低24位为网卡序列号
    2. MAC帧格式：|前导码（48）|目的MAC地址(48)|源(MAC)地址（48）|类型（16）|数据（46B~1500B）|FCS校验和(32)|
      1. 46B如何得到？最短帧长为64B，而首部和尾部共需要18B(6+6+2+4)，故最短数据段长度为46B
    3. 静态MAC相同时，设备都不能正常通信。
    4. 此前还有LLC为帧加序号
  4. 高速以太网
    1. 100BASE-T以太网
    2. 吉比特以太网
      1. 两个标准IEEE 802.3z(光纤)，IEEE 802.3ab(UTP5类线)
      2. 支持半双工（CSMA/CD）与全双工两种工作方式
    3. 10吉比特以太网必须工作在全双工模式下, 不使用CSMA/CD, 不使用广播链路
  5. 以太网的特点
    1. 广播机制：在以太网中，如果一个节点要发送数据，那么它将以广播方式把数据通过总线发送出去，连在总线上的所有节点（包括自身）都能听到发送节点发送的数据信号

3.6.3 IEEE 802.11 无线局域网

无线局域网的组成

  1. 有固定基础设施无线局域网
    1. 使用星形拓扑结构，中心称为AP(接入点，又可称为基站)
    2. 最小构件是基本服务集BSS：包含一个接入点和若干移动站，BSS的内外通信均要通过接入点中介。
  2. 无固定基础设施无线局域网
    1. 没有上述的所谓AP,网络结构中的移动站都可以作为转发节点。
  3. 802.11局域网的MAC帧
  4. 发送数据的过程

    1. 去往AP

      1. 地址1为AP的MAC，地址2为自身的地址，地址3为目的地址

    2. 来自AP

      1. 地址1为目的MAC，地址2为AP的地址，地址3为发送地址

​ 无线局域网由于接收信号的强度往往远小于发送信号的强度，故如果要实现碰撞检测需要更大的消费

3.6.4 VLAN基本概念与基本原理

虚拟局域网

动机：

  • 单一广播域:
    • 所有第2层广播流量(ARP, DHCP, 未知目的MAC地 址位置)必须穿越整个LAN
    • 安全/隐私、效率问题

基本思想：

把一个较大的局域网分割成较小的与地理位置无关的逻辑上的虚拟局域网，这样的每个虚拟局域网就是一个较小的广播域

分割标准

  • 基于交换机端口
  • 基于网卡地址
  • 基于网络层地址

802.3ac标准下的VLAN

在以太网帧中插入一个4字节的标识符，用于标识属于哪个VLAN

3.7 广域网

3.7.1 广域网基本概念

广域网由若干节点交换机以及连接交换机的链路组成，重在资源共享

3.7.2 PPP协议

某个HDLC协议已经被淘汰了

使用串行线路通信的面向字节的协议。该协议用在直接连接的两个节点的链路上，实现点对点的链接前身是SLIP协议

三个组成部分

  1. 链路控制协议(LCP)

    1. 建立数据链路：在建立状态阶段协商数据链路协议的选项
    2. 配置、测试和管理数据链路，身份验证

  2. 网络控制协议(NCP)

    1. 为网络层协议建立和配置逻辑连接

  3. 封装ip数据报到串行链路单元(MTU)

PPP帧格式（字节为单位）：**|标志字段[7E]（1）|地址字段[FF]（1）|控制字段[03]（1）|协议（2）|IP数据报（<=1500）|FCS(2)|尾部（1）|**

PS:如果标志字段7E出现在数据报字段中，需要对数据报字段中的出现进行字节填充，将信息字段中出现的0x7E 字节转变成为2字节序列(0x7D, 0x5E) 若信息字段中出现一个0x7D 字节, 则将其转变成为2字节序列 (0x7D, 0x5D)

PPP协议工作过程（运行状态变化）

初始状态为链路静止状态

注意事项

  1. PPP提供差错检测但不提供纠错功能，只保证无差错接收，是不可靠的传输协议，因此也不使用序号和确认机制
  2. 仅支持点对点的链路通信，不支持多点线路
  3. 只支持全双工链路
  4. ppp的两端可以运行不同的网络层协议，但仍然可以使用同一个ppp进行通信
  5. ppp是面向字节的

3.8.1网桥

3.8.2 局域网交换机

  1. 交换机的原理和特点

    1. 原理：本质上是一个多端口的网桥，分割碰撞域
    2. 特点：
      1. 以太网交换机的每个端口都直接与单台主机相连，并且一般工作在全双工方式
      2. 以太网交换机能同时连接多对端口，使每对相互通信的的主机仿佛独占通信一样无碰撞地传输数据
      3. 以太交换机是一种即插即用设备，其内部的帧的转发是通过自学习算法自动地逐渐建立起来的
      4. 以太交换机使用专用的交换结构芯片，交换速率较高
        1. 存储转发
        2. 直通交换 接收到帧的目的地址（不含前导码的前6B）即可开始转发
        3. 无碎片模式 接收到前64B即可开始转发
      5. 以太交换机独占传输媒体的带宽

  2. 交换机的自学习功能

    1. 交换机收到一个帧时，查看帧的目的和源地址是否在转发表内，如果在，则直接发送，如果不在，则将源地址直接添加进转发表，同时通过洪泛（不包括源地址端口），获取到目的地址的信息
    2. 洪泛：从中心主机向所有的端口发送信息（与广播并不相同）
