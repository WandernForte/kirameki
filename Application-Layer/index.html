6.1 网络应用体系结构

  • 客户机/服务器结构(C/S)

    • 服务器
    • 客户机
      • 不会与其他客户机直接通信

  • 点对点结构 (peer-peer,p2p)

    • 无永远在线的服务器
    • 任意端系统/节点之间可以直接通讯
    • 节点间歇性地接入网络
    • 节点可能改变ip地址
    • 高度可压缩但难于管理

  • 混合结构

    • Napster
      • 文件传输用p2p结构
      • 文件搜索用C/S结构

6.2 网络应用通信原理

进程间通信

不同主机上运行的进程间通信[客户进程:发信，服务进程:等待]即便是p2p结构，也存在服务进程与客户进程的分异

  • 消息交换

套接字

  • 进程间利用socket实现消息交换， 发送方将消息送到接口处
  • 发送方依赖传输基础设施将消息传到接收方所在主机，送到接收方的接收接口处
    • 传输基础设施提供：传输协议的选择， 参数的选择
  • 接收方从接口获取信息

寻址进程

  • 每个进程拥有确定的标识符
    • ip地址+端口号

应用层协议

  • 公开协议

协议内容

  • 消息类型
  • 语法格式
  • 字段语义
  • 规则
    • 进程何时发送/响应消息
    • 进程如何发送/响应消息

应用对传输服务的需求

  • 可靠性
  • 延迟
  • 带宽

实际提供的服务

  • TCP
  • UDP

6.3 域名解析系统(DNS)

Domain Name System 用于构建域名和ip地址间的映射

  • 本质是一个分布式数据库
  • 功能是完成域名的解析（映射）

域名是一级一级索引得到的

例如：客户想要访问www.amazon.com

  • 客户端查询根服务器，找到com域名解析服务器
  • 再查询com域名解析服务器，找到amazon.com域名解析服务器
  • 再查询amazon.com域名解析服务器，获得 www.amazon.com 的ip

其他类型：本地DNS

定义XXDNS=XX域名服务器

  • 不严格属于上述层级体系
  • 每个ISP至少有一个本地DNS
  • 当主机需要DNS查询时，首先访问本地DNS
  • 本地DNS无法解析域名时，再查询根域名解析服务器（层级查询）

DNS记录

存在域名服务器数据库中的内容

  • 资源记录 RR(name, value, type, ttl, class) type决定name和value的值
    • type=A, name=主机域名, value=ip地址
    • type=NS, name=域, value=该域权威DNS的主机域名
    • type=CNAME, name=某一真实域名的别名, value=真实域名
    • type=MX, name=邮件服务器别名, value=邮件服务器域名

DNS协议与信息

header(12 byte)

  • id: 16bits, 同一查询/回复过程消息使用同一ID
  • flags:
    • 查询或回复
    • 期望递归
    • 递归可用
    • 权威回答

FTP

文件传输协议

分离控制，数据连接

  • 客户机通过TCP与服务器连通
  • 客户机通过控制连接进行授权
  • 客户机通过控制连接访问服务器的目录
  • 服务器接收到传输请求后。打开另一条TCP数据链路连接客户端
    • 服务器维护这样的状态变化：当前目录， 预认证
  • 传输完毕关闭数据链路
